2. FELMERÜLT PROBLÉMÁK ÉS MEGOLDÁSAIK               (~3 perc)
   Előadó: mindketten (mindenki a saját részét meséli)
------------------------------------------------------------
PROBLÉMA 1 – Képfeltöltés és tárolás (Patrik meséli):
  → Gond: A feltöltött képeket nem lehetett egyszerűen kiszolgálni
    a Laravel alapértelmezett private storage mappájából.
  → Megoldás: `php artisan storage:link` paranccsal public storage
    szimlinket hoztunk létre, a képek a `public/storage/report_images/`
    mappába kerülnek, így közvetlenül URL-ből elérhetők.

PROBLÉMA 2 – Térkép integráció (Odett meséli):
  → Gond: A Google Maps API kulcs igénylése és limitált ingyenes kvóta
    miatt nem volt megvalósítható az eredeti terv.
  → Megoldás: Leaflet.js + OpenStreetMap tile-ok – teljesen ingyenes,
    nem kell API kulcs, jól integrálható Angular komponensbe.
    Két módban működik: csak megjelenítő (readonly) és interaktív
    (kattintásra koordináta visszaadása a form-nak).

------------------------------------------------------------
3. MIT TANULTUNK A PROJEKT ALATT?                    (~2 perc)
   Előadó: mindketten (röviden, 2-3 mondat fejenként)
------------------------------------------------------------
Odett:
  - Angular 19 standalone komponens architektúra és OnPush
    change detection a teljesítmény javítása érdekében.
  - Leaflet.js integrálása Angular NgZone-nal, hogy a térkép ne
    akassza az Angular változásdetekciót.
  - Problémamegoldó képesség: ha egy eszköz nem működik
    (Google Maps), meg kell találni az alternatívát.

Patrik:
  - Laravel Sanctum Bearer token alapú autentikáció felépítése.
  - Middleware lánc tervezése (auth → check.banned → admin),
    szerepkör alapú hozzáférés-vezérlés.
  - Soft delete alkalmazása az adatintegritás megőrzéséhez.
  - Önálló problémamegoldás: API tesztelés Postmannel,
    automatizált feature tesztek írása.

5. WEBOLDAL BEMUTATÁSA ÉLŐBEN                       (~11 perc)
   (böngésző, futó alkalmazás – kód nem látható)
------------------------------------------------------------

  -- Odett mutatja: FRONTEND                          (~6 perc)

     1. Főoldal (/)
        - Slider: top 3 leghitelesebb bejelentés automatikus váltással
        - Szűrő panel: kategória, dátum intervallum, rendezési szempontok
        - Bejelentés kártyák, lapozás (9 / oldal)

     2. Bejelentés részlete (/reports/:id)
        - Összes adat megjelenítése (cím, leírás, kategória, dátum, tanúk)
        - Leaflet térkép helyszínnel (readonly mód)
        - Upvote / downvote szavazás (visszavonás, felülírás)
        - Képgaléria lightbox nézettel (nyíl, ESC)

     3. Új bejelentés létrehozása (/reports/create)
        - Kategória, cím, leírás, dátum, tanúk száma mezők
        - Interaktív Leaflet térkép – kattintásra koordináta beállítás
        - Képfeltöltés előnézettel (max 10 db, max 5 MB/kép)

     4. Profil oldal (/profile)
        - Felhasználói adatok, szerkesztés (név, e-mail, jelszó)
        - Saját bejelentések listája státuszjelzőkkel

     5. Statisztikák (/statistics)
        - Összesítők, státusz bontás, kategória progress bar,
          top 5 leghitelesebb, legutóbbi 5 bejelentés

  -- Patrik mutatja: BACKEND MŰKÖDÉSE az élő oldalon   (~5 perc)
     (azt mutatjuk meg a weboldalon keresztül, ami a backend logikáját tükrözi)

     1. Regisztráció és bejelentkezés
        - Validáció: hibás adatok esetén hibaüzenet mezőnként
        - Sikeres belépés után Navbar változik (profil, kijelentkezés megjelenik)
        - Tiltott felhasználó: 403 üzenet jelenik meg

     2. Bejelentés létrehozása → státusz: „Várakozik"
        - Új bejelentés feltöltése után a státusz „pending" – admin jóváhagyásra vár
        - A főoldalon nem látszik, amíg nincs jóváhagyva

     3. Admin panel – moderáció (/admin/reports)
        - Bejelentések szűrése státusz szerint
        - Jóváhagyás → megjelenik a főoldalon
        - Elutasítás → nem kerül ki

     4. Admin panel – felhasználókezelés (/admin/users)
        - Keresés névre / e-mailre valós időben
        - Tiltás / tiltás feloldása

     5. Admin panel – kategóriák (/admin/categories)
        - Kategória létrehozása, szerkesztése, törlése
        - Az új kategória azonnal megjelenik a szűrőben és az űrlapon
